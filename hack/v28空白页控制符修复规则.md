# v28 空白页控制符修复规则

## 控制符结论

当前文本表中的 `{CTRL_0001}` 会造成换页/换行类行为。若它前后没有可见文字，就会在运行时表现为需要按键确认的空白对白框或空白说明页。

本轮确认需要统一处理的风险形态：

- 文本开头是 `{CTRL_0001}`：首个页面为空。
- 文本结尾是 `{CTRL_0001}`：末尾多出空白页。
- 文本中出现 `{CTRL_0001}{CTRL_0001}`：中间多出空白页。
- 两个 `{CTRL_0001}` 之间只有颜色/样式控制符，例如 `{CTRL_0103}`、`{CTRL_0000}`，没有可见文字。

## 扫描规则

不能在最终替换字节里直接搜索 `01 00` 判断换页。

原因：控制符是小端 16 位值，`{CTRL_0103}{CTRL_0000}` 的字节序列为：

```text
03 01 00 00
```

中间跨字节会出现 `01 00`，但这不是独立的 `{CTRL_0001}`。因此空页审计必须基于文本控制符 token，即 `{CTRL_0001}`，不能基于裸字节滑动匹配。

## 写回规则

写回前对 `message` 文本执行空页 trim：

1. 按 `{CTRL_0001}` 分页。
2. 去掉没有可见字符的页面。
3. 空页面里的非 `{CTRL_0001}` 控制符不丢弃，移动到相邻可见页。
4. 固定 `CTRL_0000` 子槽位仍保持原始子槽位边界和偏移，只在子槽内部清理空白 `{CTRL_0001}` 页。
5. 固定子槽位核对非换页控制符序列，避免颜色、样式、变量控制符被误删或重排。
