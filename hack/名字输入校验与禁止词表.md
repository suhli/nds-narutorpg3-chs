# 名字输入校验与禁止词表

## 结论

`msg/wifi/kinshi_msg.msg` 不是普通显示文案，而是名字/好友登记流程使用的禁止词表。

该文件包含一个大的 `{CTRL_0000}` 分隔词表，例如：

```text
ぶっとばす{CTRL_0000}げんばく{CTRL_0000}...
```

它必须暂时保留原版 CP932 字节。全量文本回写如果把它翻译成中文码点，会污染名字校验输入路径，导致有效名字也可能触发：

```text
そのなまえは　とうろくできません！{CTRL_0001}べつのなまえを　つけてください
```

对应当前汉化显示：

```text
这个名字无法登录！{CTRL_0001}请使用其他名字。
```

## 当前处理

全量文本回写默认排除：

```text
msg/wifi/kinshi_msg.msg
```

相关脚本：

```text
tools/build_text_writeback_smoke_rom.py
tools/build_full_writeback_menu_overlay_rom.py
```

已构建修正版：

```text
rom/narutorpg3_chs_full_writeback_menu_v2.nds
```

静态校验显示 v2 的 `kinshi_msg.msg` 与原版 unpacked 文件 SHA-1 一致。

## 后续方案

如果后续要支持中文玩家名或改造名字输入法，需要单独研究名字校验函数和禁止词表格式。届时不能直接翻译旧表，而应：

- 确认输入缓冲区编码。
- 确认禁止词表遍历方式和终止条件。
- 为新编码建立独立禁止词表或 patch 校验逻辑。
- 保留原版日文输入路径的兼容性。
