# 一、0x06880000 是什么区域

这是 **NDS VRAM 映射区，不是普通 RAM**

```
0x06800000–0x0689FFFF  VRAM-A/B/C/D 映射区
```

你现在用的是：

```
0x06880000
```

这是：

```
VRAM-C 或 VRAM-D 的 BG tile / OBJ tile 区域
```

用于：

```
字体tile图形
```

也就是：

```
.chr → tile像素.tbl → tile索引映射.plt → palette
```

---

# 二、FUN\_overlay\_0\_\_02086870 的完整作用

这个函数：

```
加载 NitroFS 字体文件 → 拷贝到 VRAM → 设置 tbl 指针
```

加载顺序：

```
font/font_1x1.chr → VRAMfont/font_1x2.chr → VRAMfont/font_1x1.tbl → RAMfont/font_1x2.tbl → RAMfont/font.plt     → RAM
```

关键调用：

```
bl FUN_overlay_0__0207f80c
```

这个函数是：

```
NitroFS 文件读取函数
```

# 四、VRAM大小怎么看

NDS VRAM bank大小固定：

```
VRAM-A = 128 KBVRAM-B = 128 KBVRAM-C = 128 KBVRAM-D = 128 KB
```

你现在用：

```
0x06880000
```

通常映射：

```
VRAM-C 128 KB
```

范围：

```
06880000–0689FFFF
```

大小：

```
128 KB
```

# 五、如何扩展字体

你有 3 种方法：

---

## 方法 1（推荐）：替换 NitroFS 文件（无需改代码）

直接替换：

```
NitroFS/font/font_1x1.chrNitroFS/font/font_1x2.chrNitroFS/font/font_1x1.tblNitroFS/font/font_1x2.tblfont/font.plt
```

只要：

```
新文件 size ≤ VRAM空间
```

无需修改任何代码。

---

## 方法 2：修改 NitroFS 路径（加载新文件）

修改：

```
0208689c  ldr r0,=020AD42C
```

指向：

```
"font/myfont.chr"
```

替换字符串即可。

---

## 方法 3：改 VRAM地址（支持更大字体）

修改：

```
DAT_overlay_0__020869e8 = 06880000
```

改为：

```
06840000
```

或：

```
06800000
```

扩大空间。
