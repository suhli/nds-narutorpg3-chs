# 剧情消息扩展码段兼容性

## 现象

`0xF040..0xFAAB` 码表在标题/菜单 overlay 文本中可以显示，但进入新建存档后的早期剧情时，文本会自动跳过且显示异常。

## 判断

菜单绘制链路和剧情消息解释链路不能视为同一个兼容面。`F0 xx` 形状虽然满足 Shift-JIS trail 规则，也能被菜单绘制路径使用，但剧情消息解释器可能把 `F0` 作为特殊字节或状态入口处理。

因此，旧结论需要收窄为：

```text
0xF040..0xFAAB 已验证可用于标题/菜单 overlay 初步显示；
尚未验证可用于剧情 message stream；
剧情自动跳过后，应视为存在高风险。
```

## 8840 段试验

已生成一套低段 SJIS 形状码表：

```text
plan/cache/text-writeback-smoke/code-table-safe-8840/zh_code_table.tsv
```

分配范围从 `0x8840` 开始，避开当前 raw text word 冲突，1986 项，collision count 为 0。

对应试验 ROM：

```text
rom/narutorpg3_chs_full_writeback_menu_safe8840_no_friend_kinshi_v1.nds
```

同时生成 `0x9040` 起始的备选试验 ROM：

```text
rom/narutorpg3_chs_full_writeback_menu_safe9040_no_friend_kinshi_v1.nds
```

`0x9040` 版本用于排除 `0x8840` 可能被原字体已有字符抢先命中的干扰。

如果 `0x8840` 或 `0x9040` ROM 的剧情不再自动跳过，应把中文扩展码默认段从 `0xF040` 迁移到低段 SJIS 形状区间，并重新生成全量码表、字体文件、菜单 overlay 编码和文本写回预览。

## 后续注意

`0x8840` 仍需运行时确认字体 hook 是否优先命中新字模。如果它被原字体已有字符抢先命中，需要继续选择更靠后的低段 SJIS 形状范围，或调整绘制 hook 的优先级。
