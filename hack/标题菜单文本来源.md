# 标题菜单文本来源

更新时间：2026-06-02

## 结论

标题红色菜单中的可见选项文本不是 `data/text/msg/...` 下的普通消息文件，而是硬编码在 ARM9 overlay：

```text
overlay/overlay_0004.bin
```

因此当前 `text/writeback/encoded_preview.tsv` 的 5863 条全量固定槽写回没有覆盖标题菜单选项。全量写回 ROM 中这些 overlay 字符串仍与原版一致。

## 已确认字符串

在原版解包目录和全量写回解包目录中均能找到以下 CP932 字符串：

```text
overlay/overlay_0004.bin
0x5270 slot=0x10  　はじめから
0x5280 slot=0x18 　ＷｉーＦｉせってい
0x5298 slot=0x14 　ともだちにくばる
0x52AC slot=0x10 　つづきから
0x52BC slot=0x10 　はじめから
0x52CC slot=0x14 　ファイルさくじょ
0x52E0 slot=0x18 　ＷｉーＦｉせってい
0x52F8 slot=0x14 　ともだちにくばる
```

这些槽位是零填充的固定长度字符串。当前探针只做等长或更短替换，不改变 overlay 大小。

## 与字体系统的关系

全量写回 ROM 的原始字体文件仍与原版相同：

```text
data/font/font_1x1.tbl
data/font/font_1x2.tbl
```

当前字体集成方式不是直接替换原 `.tbl/.chr` 成大中文字库，而是增加动态缓存文件并 patch `arm9.bin` 与 `overlay_0000.bin`：

```text
data/font/chs_1x1.map
data/font/chs_1x1.chunk
data/font/chs_1x2.map
data/font/chs_1x2.chunk
```

所以标题菜单日文不变的主因是菜单 overlay 文本没有纳入 dump/写回，而不能直接证明动态字体补丁未生效。

## 探针 ROM

已构建一个只替换标题菜单 overlay 字符串的探针 ROM：

```text
rom/narutorpg3_chs_full_writeback_menu_overlay_probe.nds
rom/unpacked/narutorpg3_chs_full_writeback_menu_overlay_probe/
plan/cache/text-writeback-smoke/menu-overlay-probe-records.json
```

替换内容：

```text
　はじめから              -> 　开始
　ＷｉーＦｉせってい      -> 　通信设置
　ともだちにくばる        -> 　分发好友
　つづきから              -> 　继续
　ファイルさくじょ        -> 　删除存档
```

探针用途：

- 如果标题菜单能显示这些中文，说明菜单绘制路径可以复用当前中文码表/动态字体，后续需要把 overlay 字符串纳入 dump 和写回。
- 如果标题菜单显示空白、乱码或缺字，说明标题菜单绘制路径与正文不同，需要单独追踪该 overlay 的绘制函数和字形查找路径。
