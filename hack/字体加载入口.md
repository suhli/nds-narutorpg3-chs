# 一、0x06880000 是什么区域

这是 **NDS VRAM 映射区，不是普通 RAM**

```
0x06800000–0x0689FFFF  VRAM-A/B/C/D 映射区
```

你现在用的是：

```
0x06880000
```

这是：

```
VRAM-C 或 VRAM-D 的 BG tile / OBJ tile 区域
```

用于：

```
字体tile图形
```

也就是：

```
.chr → tile像素.tbl → tile索引映射.plt → palette
```

---

# 二、FUN\_overlay\_0\_\_02086870 的完整作用

这个函数：

```
加载 NitroFS 字体文件 → 写入字体区 → 设置 chr/tbl/plt 指针和计数
```

加载顺序：

```
font/font_1x1.chr → VRAM
font/font_1x2.chr → VRAM
font/font_1x1.tbl → VRAM
font/font_1x2.tbl → VRAM
font/font.plt     → RAM/heap
```

关键调用：

```
bl FUN_overlay_0__0207f80c
```

这个函数是：

```
NitroFS 文件读取函数
```

---

# 三、VRAM结构体地址

```
DAT_020B73E0
```

是：

```
字体系统全局结构体
```

结构：

```
struct FontContext {
    u32 chr1x1_vram;      // +0x00 → 06880000
    u32 chr1x2_vram;      // +0x04 → 06881C00
    u32 plt_ptr;          // +0x08 → RAM/heap
    u32 plt_count;        // +0x0C
    ...
    u32 tbl1x1_vram;      // +0x20 → 0688E6C0
    u32 tbl1x1_count;     // +0x24 → 000000E0
    u32 tbl1x2_vram;      // +0x28 → 0688EA40
    u32 tbl1x2_count;     // +0x2C → 0000032B
}
```

# 四、VRAM大小怎么看

NDS VRAM bank大小固定：

```
VRAM-A = 128 KBVRAM-B = 128 KBVRAM-C = 128 KBVRAM-D = 128 KB
```

你现在用：

```
0x06880000
```

通常映射：

```
VRAM-C 128 KB
```

范围：

```
06880000–0689FFFF
```

大小：

```
128 KB
```

# 五、如何扩展字体

你有 3 种方法：

---

## 方法 1（仅作对照验证）：替换 NitroFS 文件（无需改代码）

直接替换：

```
NitroFS/font/font_1x1.chrNitroFS/font/font_1x2.chrNitroFS/font/font_1x1.tblNitroFS/font/font_1x2.tblfont/font.plt
```

只要：

```
新文件 size ≤ VRAM空间
```

无需修改任何代码。

---

## 方法 2：修改 NitroFS 路径（加载新文件）

修改：

```
0208689c  ldr r0,=020AD42C
```

指向：

```
"font/myfont.chr"
```

替换字符串即可。

---

## 方法 3：改 VRAM地址（支持更大字体）

修改：

```
DAT_overlay_0__020869e8 = 06880000
```

改为：

```
06840000
```

或：

```
06800000
```

扩大空间。

# 六、真正绘制使用的是 tbl 指针

在：

```
DAT_020B73E0 + 0x20 → font_1x1.tbl 指针
DAT_020B73E0 + 0x24 → font_1x1.tbl 项计数
DAT_020B73E0 + 0x28 → font_1x2.tbl 指针
DAT_020B73E0 + 0x2C → font_1x2.tbl 项计数
```

绘制函数会：

```
读取tbl → 查tile index → 从 VRAM tile读取 → 绘制
```

当前已确认 `020866BC` 会对 4 字节 tbl 表项做二分查找。

# 九、你已经完全找到字体系统结构

真正结构：

```
NitroFS
  ↓ FUN_overlay_0__0207f80c
  ↓ 加载 .chr → VRAM
  ↓ 加载 .tbl → VRAM
  ↓ 加载 .plt → RAM/heap
DAT_020B73E0 保存 chr/tbl/plt 指针和 tbl 项计数
```

不是：

```
不是 hash
不是全量中文字库常驻 VRAM 的最终方案
```

是：

```
0208671C 通过 020866BC 对 tbl 做二分查找，得到 glyph index，再计算 chr_base + unit_size * glyph_index
```
