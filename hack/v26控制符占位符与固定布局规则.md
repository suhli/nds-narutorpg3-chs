# v26 控制符、占位符与固定布局规则

## 结论

本轮新增的乱码和自动结束问题不是字体问题，而是可见文本中残留了单字节 ASCII。该类字节在 NDS 文本流里可能被解释为控制参数、SJIS 尾字节或脚本流的一部分，导致渲染提前结束、颜色没有复位，或后续文本全部变色。

当前规则：除已确认的结构尾部参数外，写回器必须把控制符外的可见 ASCII 转为全角字符或中文短译。

## 已确认案例

- `zh_txt_dc122c8a_000C2C_0041`：`Y按钮` 中的 `Y` 必须写为全角 `Ｙ`。它位于 `CTRL_0103/CTRL_0002` 高亮段内，半角 `59` 会破坏后续高亮复位。
- `msg/item_msg.msg`：`少量恢复1人的体力` 中的 `1` 必须写为 `１`。
- `msg/wifi/friend_msg.msg`：`确认/修改一句话评论` 中的 `/` 必须写为 `／`。
- `zh_txt_de7d406d_000124_0004`：误抽出的半角引号和感叹号应恢复为原始含义 `「！」`。

## 可保留的单字节尾部

下列单字节不是可见文本，属于已确认的结构尾部参数：

```text
zh_txt_bb92057e_000008_0000  ... {CTRL_0008}{CTRL_0000}N
zh_txt_b91ed4cf_000008_0000  ... {CTRL_0008}{CTRL_0000}4{CTRL_0000}{CTRL_5400}{CTRL_0000}{CTRL_7C00}
```

写回器应仅保留这些尾部参数；尾部之前的可见文本仍要做全角化。

## 控制符作用归纳

- `CTRL_0001`：正文中的换行/换页类控制。对控制位敏感的对白，应按原始控制位分段写回。
- `CTRL_0000`：在合并说明表中是固定子槽位边界；在高亮段中也可作为结束参数的一部分。不能随意移动。
- `CTRL_0103 + CTRL_0002`：高亮/颜色开始；后续 `CTRL_0103 + CTRL_0000` 结束并复位。高亮段内的可见文字不能用半角 ASCII。
- `CTRL_0008`：在少数大表尾部后接结构参数，当前确认会跟 `N` 或 `4` 等单字节尾参。
- `03 00`：普通 message 结束符。剧情对白默认保留原始位置；菜单/好友说明类短文本可提前结束。
- `00 00 00 00`：部分说明表的真实 NUL4 结束符。已知说明表应在译文后立即写入 NUL4，再用 `00` 填尾。

## overlay 固定布局

overlay 中同一条字符串包含多个标签或依赖原始间距时，不能短译后直接 `00` 填尾。应按原始可见宽度用全角空格补齐，再填槽尾 `00`。

v26 已改为固定宽度覆盖的代表行：

```text
menu_overlay_0003_0442A0  攻击 / 防御 / 速度
menu_overlay_0000_030100  提升 / 相同 / 下降 / 装备中
menu_overlay_0000_0301F8  当前经验值
menu_overlay_0000_030214  距下一级还差
menu_overlay_0002_0128B8  获得了！
menu_overlay_0003_045228  通灵纸暗号提示
menu_overlay_0003_0452B8  存档文件·临
menu_overlay_0003_0452D8  存档文件·兵
```

## 审计规则

`tools/audit_v24_structural_risks.py` 现在会额外检查：

- 控制符外是否仍有可见 ASCII。
- 固定宽度 overlay 候选是否仍未覆盖。
- 合并 `CTRL_0000` 子槽位、提前 `03 00`、提前 NUL4 等既有结构规则是否还有遗漏。

v26 审计结果：文本风险 0，overlay 风险 0；仅保留 2 条已确认结构尾部 ASCII 参数。
