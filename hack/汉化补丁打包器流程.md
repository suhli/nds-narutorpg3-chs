# 汉化补丁打包器流程

`patcher/patcher.py` 是当前全量文本回写和菜单回写基线的冻结打包入口。

## 打包流程

1. 检查 `rom/origin.nds`，必要时解包到 `patcher/work/origin_unpacked`。
2. 准备文本素材。
   默认模式使用 `patcher/resources/text/encoded-preview-struct-adjusted.tsv` 作为结构安全译文预览。
   替换译文时，从冻结译文 TSV 重新生成码表、字体 manifest、编码预览、结构审计结果和菜单翻译表。
3. 从 TTF 生成动态中文字库 payload。
   输出文件包括 `chs_1x1.map`、`chs_1x1.chunk`、`chs_1x2.map`、`chs_1x2.chunk`。
4. 修改 ROM payload。
   写入 data/text 文本、菜单 overlay 文本、动态字体资源和当前字体 hook。
5. 使用 `patcher/tools/ndstool.exe` 重新打包为新的 ROM，并执行 `ndstool -i` 校验。

## 输入资源

- `patcher/resources/text/frozen_translation.tsv`：冻结译文表。
- `patcher/resources/text/zh_code_table.tsv`：当前中文字码表。
- `patcher/resources/text/font_manifest.json`：字模生成 manifest。
- `patcher/resources/text/encoded-preview-struct-adjusted.tsv`：已做结构审计和自动调整的写回预览。
- `patcher/resources/text/translation-struct-manual-overrides.tsv`：结构审计中的手工覆盖规则。
- `patcher/resources/menu/overlay_menu_translations.tsv`：菜单 overlay 翻译表。
- `patcher/resources/fonts/1x1.ttf`：默认 1x1 字体源文件。
- `patcher/resources/fonts/1x2.ttf`：默认 1x2 字体源文件，当前内容来自 `MZPXorig.ttf`。

## 工具打包

根目录 `tools/` 被 `.gitignore` 忽略，所以 patcher 自带一份最小工具闭包：

- `patcher/tools/*.py`：打包流程依赖的 20 个 Python 脚本。
- `patcher/tools/ndstool.exe`：解包、重打包和 ROM 信息校验工具。

运行时优先使用 `patcher/tools/`。只有开发环境缺少 bundled tools 时，才回退到根目录 `tools/`。

## 重建模式

默认重建：

```powershell
.\.venv\Scripts\python.exe patcher\patcher.py --output rom\narutorpg3_chs_patcher.nds
```

替换字体后一键重建：

```powershell
.\.venv\Scripts\python.exe patcher\patcher.py --font path\to\font.ttf --output rom\narutorpg3_chs_font_test.nds
```

分别指定 1x1 和 1x2 字体：

```powershell
.\.venv\Scripts\python.exe patcher\patcher.py --font-1x1 path\to\8px.ttf --font-1x2 path\to\16px.ttf --output rom\narutorpg3_chs_font_test.nds
```

替换冻结译文后一键重建：

```powershell
.\.venv\Scripts\python.exe patcher\patcher.py --translation-table path\to\frozen_translation.tsv --output rom\narutorpg3_chs_text_test.nds
```

如果直接修改了 `patcher/resources/text/frozen_translation.tsv`：

```powershell
.\.venv\Scripts\python.exe patcher\patcher.py --rebuild-text-assets --output rom\narutorpg3_chs_text_test.nds
```

使用已生成的编码预览重建：

```powershell
.\.venv\Scripts\python.exe patcher\patcher.py --translation-preview path\to\encoded_preview.tsv --output rom\narutorpg3_chs_preview_test.nds
```

## 安全规则

- `rom/origin.nds` 只作为输入读取，不能覆盖、原地 patch 或作为输出路径。
- 如果指定的输出 ROM 已存在，patcher 会自动改用带时间戳的同名旁路输出，避免覆盖旧结果。
- 每次构建都会写入 `patcher/work/build_*`，其中包含日志、生成的文本素材、生成的字体 payload、ROM 工作目录和构建摘要。
- 遇到缺字不会中止构建。patcher 会记录文本编码缺字、菜单编码缺字和 TTF 字形缺失，构建结束后按 text、menu、font 分组打印缺字；字体缺字会再按 1x1、1x2 分组并打印对应 TTF 路径。报告会写入 `missing-chars-report.json`、`missing-chars.tsv` 和 `font-missing-chars.tsv`。
- `missing-chars.tsv` 是总表；字体缺字行会列出 `mode`、码表 `code` 和实际缺字的 TTF 路径，便于判断需要替换 1x1 字体还是 1x2 字体。
- 替换译文重建时，结构审计硬错误、控制符不匹配、固定槽溢出或非缺字导致的菜单翻译错误仍会中止构建。

## 1x2 字模处理

早期 1x2 字模使用 16x16 渲染后横向压缩到 8x16，游戏内显示会显得笔画过密。

当前 patcher 改为按 8x12 目标格直接栅格化，再垂直居中放入 8x16 双 tile。这个逻辑更接近 1x1 字模的目标格处理方式，不再做 16 宽到 8 宽的横向挤压。
