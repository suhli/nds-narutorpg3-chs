# 汉化补丁打包器流程

`patcher/patcher.py` is the current frozen build orchestrator for the full writeback/menu baseline.

Pipeline:

1. Unpack `rom/origin.nds` into `patcher/work/origin_unpacked` when the unpacked origin is missing.
2. Prepare text assets:
   - default: use the frozen structure-safe preview in `patcher/resources/text/encoded-preview-struct-adjusted.tsv`;
   - replacement translation: rebuild code table, font manifest, encoded preview, structure audit output, and menu translation table from a frozen translation TSV.
3. Build dynamic Chinese font payload files from TTF:
   - `chs_1x1.map`
   - `chs_1x1.chunk`
   - `chs_1x2.map`
   - `chs_1x2.chunk`
4. Patch text payloads, menu overlay payloads, and current dynamic font hooks.
5. Repack a new ROM and verify it with `tools/ndstool.exe -i`.

ROM safety rule: `rom/origin.nds` is only read as input and is never used as an output path.

When a requested output ROM already exists, the patcher selects a timestamped sibling output path before invoking the lower-level ROM builder. Menu rows with `pending_font_chars` are treated as build blockers during translation rebuilds.
